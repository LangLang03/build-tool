## 设计理念

build-tool 的设计遵循以下原则：

### 1. 模块化

每个模块职责单一，通过明确的接口通信：

```
utils.sh ──► output.sh ──► log.sh
    │            │           │
    └────────────┴───────────┘
                 │
            config.sh
                 │
    ┌────────────┼────────────┐
    │            │            │
cache.sh    plugin.sh     core.sh
```

### 2. 可扩展性

通过插件系统扩展功能，无需修改核心代码。

### 3. 用户友好

提供丰富的输出格式和国际化支持。

## 模块职责

### utils.sh - 工具函数

提供通用工具函数：

- **字符串处理**：trim, split, join, lower, upper
- **数组操作**：contains, unique, filter, map
- **文件操作**：ensure_dir, safe_copy, safe_move, file_hash
- **时间处理**：format_duration, format_timestamp
- **类型检查**：is_number, is_integer, is_bool
- **数学运算**：clamp, min, max, abs
- **随机生成**：random_string, uuid

### output.sh - 输出格式化

提供终端输出功能：

- **基本输出**：print, info, success, warning, error
- **格式化输出**：header, section, bullet, key_value
- **进度显示**：progress_bar, spinner
- **表格输出**：table
- **树形输出**：tree

### log.sh - 日志记录

提供日志功能：

- **多级别日志**：debug, info, warn, error, fatal
- **日志文件**：文件输出、轮转
- **日志配置**：级别设置、格式设置

### platform.sh - 平台检测

提供平台检测功能：

- **系统检测**：OS、发行版、架构
- **包管理器**：apt, dnf, pacman, brew
- **权限检测**：root, sudo

### config.sh - 配置管理

提供配置功能：

- **多格式支持**：YAML, INI, Shell
- **配置优先级**：命令行 > 环境变量 > 文件 > 默认值
- **模板处理**：变量替换

### cache.sh - 缓存管理

提供缓存功能：

- **文件哈希**：md5, sha1, sha256
- **增量构建**：变更检测
- **缓存清理**：过期、大小限制

### i18n.sh - 国际化

提供国际化支持：

- **多语言**：中文、英文
- **自动检测**：系统语言
- **插件扩展**：插件国际化

### yaml.sh - YAML 解析

提供 YAML 处理：

- **解析**：文件、字符串
- **生成**：YAML 输出
- **验证**：格式检查

### plugin.sh - 插件系统

提供插件功能：

- **插件发现**：目录扫描
- **插件加载**：source 执行
- **依赖管理**：命令、系统包
- **钩子系统**：pre_build, post_build, on_error

### core.sh - 核心引擎

提供核心功能：

- **目标管理**：注册、依赖、执行
- **依赖解析**：拓扑排序、循环检测
- **执行引擎**：顺序执行、并行执行
- **步骤管理**：step_start, step_end
- **生命周期**：init, cleanup, summary

```
    │
    ▼

    │
    ▼

```

## 初始化顺序

```bash
# 1. 加载工具函数
source lib/utils.sh

# 2. 初始化输出模块
source lib/output.sh
output_init

# 3. 初始化日志模块
source lib/log.sh
log_init

# 4. 检测平台
source lib/platform.sh
platform_detect

# 5. 加载配置
source lib/config.sh
config_init

# 6. 初始化缓存
source lib/cache.sh
cache_init

# 7. 初始化国际化
source lib/i18n.sh
i18n_init

# 8. 初始化 YAML 解析
source lib/yaml.sh

# 9. 初始化插件系统
source lib/plugin.sh
plugin_init
plugin_discover
plugin_load_all

# 10. 初始化核心引擎
source lib/core.sh
build_init
```

## 设计决策

### 为什么用 Bash？

**优点：**
- 无需安装额外依赖
- 跨平台（Linux/macOS/WSL）
- 适合系统脚本

**缺点：**
- 数据结构有限
- 错误处理脆弱
- 调试困难

**决策：** 选择 Bash 是为了最小化依赖，适合构建工具的场景。

### 为什么用关联数组？

关联数组（Bash 4.0+）提供了类似字典的功能，使配置和缓存管理更简单。

### 为什么支持多种配置格式？

- **YAML**：现代、可读性好
- **INI**：传统、简单
- **Shell**：灵活、可编程

支持多种格式让用户可以选择最适合的方式。

### 为什么用文件哈希做增量构建？

文件哈希比时间戳更可靠，不受文件系统时间影响。
